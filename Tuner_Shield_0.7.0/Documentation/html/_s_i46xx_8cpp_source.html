642  prepares the bootloader to receive a new image. After the LOAD_INIT command, using the HOST_LOAD
643  command loads the image into the device RAM. After the RAM is loaded the host issues the BOOT command.
644  When booting a FLASH image the host issues the POWER_UP command to set the system settings. Then issues
645  the FLASH_LOAD command to select and load the image from FLASH. Once the image is loaded the host sends
646  the BOOT command to boot the application. Power-up is complete when the CTS bit is set. This command may
647  only be sent while the device is powered down. Note: FLASH_LOAD is not supported in A0A or A0B revisions.

651 #endif

658  data[2] = devicePowerUpArguments.iBiasStart; //ARG3 0 IBIAS[6:0]

667  data[11] = 0; //Arg12 0
